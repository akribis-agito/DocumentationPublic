# ModeSwitchPos

Records the position when the axis enters or exits position mode.

## Overview

`ModeSwitchPos` is a read-only array that records the position feedback ([Pos](../../10-motion/01-kinematics-status/Pos.md)) captured at each transition between position mode and the current/force modes. Each element is written only at its own transition and then holds that value until the next such transition, so the two elements together give the "from" and "to" feedback of the most recent mode change. The values are latched automatically on every switching path â€” direct [GoToPosMode](GoToPosMode.md)/`GoToCurrMode`/`GoToForceMode` commands, the internal feedback-threshold switch ([PosPosFlag](PosPosFlag.md)/[PosPosTh](PosPosTh.md)), and [DInMode](../../05-inputs-outputs/04-digital-inputs/DInMode.md) digital-input switching.

## How it works

`ModeSwitchPos` is a 1-indexed array of size 2:

| Index | Latched value (`Pos` at the transition) |
|-------|------------------------------------------|
| 1     | Position when the axis **exits** position mode (i.e. enters current or force mode) |
| 2     | Position when the axis **enters** position mode |

- **Index 1** is written wherever `OperationMode` becomes current (1) or force (4): `GoToCurrMode`, `GoToForceMode`, the internal auto-switch to current/force, and the `DInMode` falling-edge switches.
- **Index 2** is written wherever `OperationMode` becomes position (3): `GoToPosMode`, the internal threshold/scheduled-end switch, and the `DInMode` rising-edge switches.

Index 1 (the force-mode entry position) is also used internally as the anchor for the position reference generated by the force loop during force-over-PIV control, which is what makes the later switch back to position mode bumpless.

## Changes between versions

In **v5 (central-i)** the position pipeline is 64-bit, so `ModeSwitchPos` stores 64-bit feedback values with the larger range shown in the frontmatter; the recording behaviour is unchanged. **v5 is central-i only**, so on standalone `ModeSwitchPos` remains the v4 32-bit value.

## Examples

```text
AModeSwitchPos[1]   ; Pos recorded when the axis left position mode (entered current/force)
AModeSwitchPos[2]   ; Pos recorded when the axis entered position mode
```
