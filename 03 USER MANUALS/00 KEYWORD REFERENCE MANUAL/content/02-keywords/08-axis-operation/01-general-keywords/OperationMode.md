# OperationMode

**Definition:**

OperationMode determines the axis’ active control mode and which of the control loops will be activated. It can be changed manually or automatically (using digital inputs, internal switching algorithm, etc.)

| OperationMode | Descriptions |
|---|---|
| 1 | **Current control mode** Only the current loop is active. Current reference can be driven by analog input, user-defined value/table or master axis (as slave drive), through CurrCmdSrc keyword. Please refer to Current operation mode for more information. |
| 2 | **Velocity control mode** Only velocity and current loops are active. Velocity reference can only be driven by analog input. |
| 3 (default) | **Position control mode** Only position, velocity and current loops are active. Position reference is generated by motion profiler, where user can use MotionMode to change the profile type. |
| 4 | **Force control mode** Under standard force control, only force and current loops are active. Under force-over-PIV control, force, position, velocity and current loops will be active, with force loop being the outermost loop generating position reference. Force reference can be driven by analog input or user-defined value/table, through ForceCmdSrc keyword. Please refer to Axis operation – Force operation mode and Control tuning – Force control for more information. |
