# OperationMode

Selects the axis control mode and which control loops are active.

## Overview

`OperationMode` determines the axis' active control mode and which control loops are activated. It can be changed manually (by direct assignment) or automatically — via digital inputs ([DInMode](../../../02-keywords/05-inputs-outputs/04-digital-inputs/DInMode.md)), the `GoTo...Mode` commands, or the internal switching algorithm using the condition-check keywords.

For graceful transitions, prefer the dedicated commands [GoToCurrMode](../03-current-operation-mode/GoToCurrMode.md), [GoToPosMode](../02-position-operation-mode/GoToPosMode.md) and [GoToForceMode](../04-force-operation-mode/GoToForceMode.md) over direct assignment, since they perform the proper preparation before switching.

## How it works

| OperationMode | Descriptions |
|---|---|
| 1 | **Current control mode** Only the current loop is active. Current reference can be driven by analog input, user-defined value/table or master axis (as slave drive), through the [CurrCmdSrc](../03-current-operation-mode/CurrCmdSrc.md) keyword. Please refer to Current operation mode for more information. |
| 2 | **Velocity control mode** Only velocity and current loops are active. Velocity reference can only be driven by analog input. |
| 3 (default) | **Position control mode** Only position, velocity and current loops are active. Position reference is generated by motion profiler, where user can use [MotionMode](../../10-motion/02-motion-configuration/MotionMode.md) to change the profile type. |
| 4 | **Force control mode** Under standard force control, only force and current loops are active. Under force-over-PIV control, force, position, velocity and current loops will be active, with the force loop being the outermost loop generating position reference. Force reference can be driven by analog input or user-defined value/table, through the [ForceCmdSrc](../04-force-operation-mode/ForceCmdSrc.md) keyword. Please refer to Force operation mode and Control tuning – Force control for more information. |

## Examples

```text
OperationMode=3     ; position control mode (default)
OperationMode=1     ; current control mode
OperationMode?      ; read the active control mode
```
