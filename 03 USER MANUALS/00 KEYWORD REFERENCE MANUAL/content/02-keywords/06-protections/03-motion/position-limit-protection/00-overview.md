# Position limit protection

Agito supports both **hardware** and **software** position limits.

**Software limits** ([FwdPLim](FwdPLim.md) / [RevPLim](RevPLim.md)) define the allowed range of motion. If a move command has a target position outside the software limits, it is rejected at [Begin](../../../10-motion/04-motion-command/Begin.md) and the rejection is logged in [ErrLog](../../../07-status-and-faults/ErrLog.md). While in motion, the distance to safely decelerate and stop without exceeding the software limit is continuously checked. If the axis is approaching a limit — in jog mode for example — the profiler decelerates and brings the axis to a stop at the limit position. The stop reason is recorded in [MotionReason](../../../10-motion/05-motion-status/MotionReason.md) (`6` = reverse software limit, `7` = forward software limit). The stop uses the emergency deceleration [EmrgDec](../../../10-motion/03-kinematics-configuration/EmrgDec.md), not the normal [Decel](../../../10-motion/03-kinematics-configuration/Decel.md).

**Hardware limits** are external signals indicating that the axis is at the end of its stroke. The configuration of limit sensors is done via [DInMode](../../../05-inputs-outputs/04-digital-inputs/DInMode.md). [LimitsStat](LimitsStat.md) reflects the statuses (triggered/not triggered) of the two limit-switch inputs (bit `0` = RLS, bit `1` = FLS). While a hardware limit sensor is triggered, a motion command in the direction of the limit is rejected as a command/message error, and the rejection is logged in [ErrLog](../../../07-status-and-faults/ErrLog.md).

While in motion, the statuses of the limit sensors are continuously checked. If the limit in the direction of motion is triggered, the profiler decelerates with [EmrgDec](../../../10-motion/03-kinematics-configuration/EmrgDec.md) and brings the axis to a stop. The stop reason is recorded in [MotionReason](../../../10-motion/05-motion-status/MotionReason.md) (`4` = reverse limit switch, `5` = forward limit switch).

The position-limit mechanism never disables the axis or raises a [ConFlt](../../../07-status-and-faults/ConFlt.md) — it only commands a controlled deceleration. It is independent of the motion-protection following-error trips ([MaxPosErr](../general-maximum-limits/MaxPosErr.md), [MaxVelErr](../general-maximum-limits/MaxVelErr.md)) and of the hardware-protection mask [ProtectMask](../../01-general-protection/ProtectMask.md).
